## Notes

The small default run before this pass was dominated by fixed overhead. The
larger hardware pass fixed the training and inference bands for DP, FSDP, and
decode, which makes the remaining failures more specific.

Open questions:

- Roofline: the measured crossover moved farther out, from `1024` on the small
  sweep to `2048` on the extended sweep, while throughput was still climbing at
  `8192`. The current crossover detector is probably measuring "first point at
  half of observed peak", not the real asymptote.
- Tensor parallel training: TP compiled many more layer collectives and landed
  `3.4x` slower than FSDP. The compute-only MFU band is missing communication
  cost for this tiny transformer shape.
- Trace: the runner now traces both FSDP and TP by default so the next Kaggle
  pass can directly explain the TP slowdown.
